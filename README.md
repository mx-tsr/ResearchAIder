# ResearchAIder
Automated agent for paper search and analysis using the AI Scientist architecture

# Как запустить локально?
## Системные требования
Для запуска проекта рекомендуется следующая конфигурация:

- Операционная система: Windows 10 или выше
- IDE: Visual Studio Code
- Python: 3.10+
- Наличие стабильного интернет-соединения
- GPU с видеопамятью 4 Гб или выше для локального запуска моделей через Ollama

#№ 1. Клонирование проекта и создание виртуального окружения

Сначала нужно открыть Visual Studio Code и терминал внутри проекта (`Terminal -> New Terminal`) и склонировать репозиторий:

```bash
git clone https://github.com/mx-tsr/ResearchAIder.git
```
Затем перейти в папку проекта (`cd ResearchAIder`) и создать виртуальное окружение:
```bash
python -m venv venv
venv\Scripts\activate
```
После активации в начале строки терминала должна появиться (venv).

Далее нужно установить необходимые зависимости:
```bash
pip install -r requirements.txt
```

## 2. Настройка OpenRouter API

Проект поддерживает использование любых API языковых моделей, но по умолчанию работает через OpenRouter API. Для создания ключа нужно перейти на сайт [OpenRouter](https://openrouter.ai/)

Затем зарегистрироваться или войти в аккаунт. Открыть раздел API Keys и создать новый API-ключ. 

При выборе модели нужно применить фильтр на бесплатные модели (free), например `deepseek/deepseek-chat-v3-0324:free`

!!! Настоятельно рекомендуется создать несколько резервных API-ключей на других аккаунтах, поскольку бесплатные модели OpenRouter имеют ограничения по количеству запросов в сутки (50  штук).

## 3. (Опционально) Настройка Ollama

Использование Ollama позволяет запускать языковые модели локально, бесплатно и без постоянного интернет-соединения.

Для того, чтобы Ollama могла использовать GPU - предварительно нужно установить CUDA с [официального сайта](https://developer.nvidia.com/cuda/toolkit). 

Затем требуется установить Ollama с [официального сайта](https://ollama.com/download/windows)

После установки необходимо убедиться, что Ollama доступна в терминале: `ollama --version`

При выборе используемой модели ее размер должен быть меньше объема видеопамяти GPU. Например: 8 GB VRAM → модели до ~7B–12B

Модель можно установить либо через приложение Ollama, либо через терминал: `ollama pull gemma4:e2b`

В настройках программы Ollama нужно выбрать контекст 16-32К для корректной обработки длинных научных публикаций.

!!! Почему рекомендуется использование Ollama даже без GPU:
- тогда агент продолжит работу даже при нестабильном интернет-соединении
- агент продолжит работу, если закончатся генерации на API ключах OpenRouter

## 4. Создание файла .env

В корне проекта создайте файл `.env`

Вставьте в него текст ниже, заменив пропуски на ваши данные:
```
OPENROUTER_API_KEY=<ваш ключ OpenRouter>
OPENROUTER_API_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_MODEL=<liquid/lfm-2.5-1.2b-thinking:free или заменить на другую бесплатную модель>

OPENROUTER_API_KEY_FALLBACK_1=<при наличии - запасной API, иначе удалите>
OPENROUTER_API_KEY_FALLBACK_2=<при наличии - запасной API, иначе удалите>
OPENROUTER_API_KEY_FALLBACK_3=<при наличии - запасной API, иначе удалите>
OPENROUTER_API_KEY_FALLBACK_4=<при наличии - запасной API, иначе удалите>
OPENROUTER_API_KEY_FALLBACK_5=<при наличии - запасной API, иначе удалите>

OLLAMA_API_BASE_URL=http://localhost:11434/v1/
OLLAMA_MODEL=<gemma4:e2b или модель, которую вы установили>

```

## 5. Запуск агента

Для запуска проекта откройте main.py и нажмите на кнопку выполнения, либо выполните в терминале: `python main.py`

После запуска система предложит ввести тему исследования на английском (связан с тем, что запросы к arxiv делаются только на анлглийском).

!!! Если вы используете Ollama, то она автоматически запустит модель, когда произойдет попытка обратится к ее API.

После этого агент автоматически:
- выполнит поиск научных публикаций, отберет 10 самых релевантных
- загрузит их PDF-файлы в директорию pdfs/, извлечет их текст в директорию txts/ и извлечет ключевую информацию в директорию extracted_info/ в формате json
- разобьет статьи на группы и сохранит разбиение в папку analysis_output в файл groups.json. Выполнит анализ по группам и сохранит каждый в .txt файл с названием group_<имя группы>.txt. 
- сгенерирует обзорную статью с промежуточными шагами, сохраняя информацию в файлы final_review_no_citations.txt - обзор без ссылок, final_review_with_citations.txt - обзор со ссылками, final_review.txt - финальный обзор с разделом гипотез
- создаст логи всей генерации в файле logs/app.log
